PowerShell — wrzucenie paczki do repo leadów

Zakładam:
- ZIP pobrany z czatu jest w: C:\Users\malim\Downloads
- lokalne repo leadflow jest w: C:\Users\malim\Desktop\leadflowv1-fresh
- branch docelowy: dev-rollout-freeze

Polecenia:

$Zip = "$HOME\Downloads\leadflow_v1_audit_fix_2026-04-18.zip"
$Temp = "$HOME\Downloads\leadflow_v1_audit_fix_2026-04-18_unpacked"
$Repo = "C:\Users\malim\Desktop\leadflowv1-fresh"
$PatchRoot = Join-Path $Temp "leadflow_v1_audit_fix_2026-04-18"

if (!(Test-Path $Zip)) { throw "Nie ma ZIP-a: $Zip" }
if (!(Test-Path (Join-Path $Repo ".git"))) { throw "Nie ma repo z .git pod: $Repo" }
if (Test-Path $Temp) { Remove-Item $Temp -Recurse -Force }

Expand-Archive -Path $Zip -DestinationPath $Temp -Force

Set-Location $Repo
git fetch origin
git checkout dev-rollout-freeze
git pull origin dev-rollout-freeze

Copy-Item "$PatchRoot\*" $Repo -Recurse -Force

git status
npm.cmd install
npm.cmd run lint
npm.cmd run build

git add .
git commit -m "Fix live calendar sync and audit V1 gaps"
git push origin dev-rollout-freeze
